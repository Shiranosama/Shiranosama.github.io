记录Windows下基于MSYS2的开发环境搭建

0. 前言

最近在折腾Windows下的开发环境，发现MSYS2是一个非常强大的工具，十分适合我这种喜欢极致轻量化的人。
做此记录，以便日后查阅。

1. 安装MSYS2

MSYS2 是一套工具和库的集合，提供构建、安装和运行原生 Windows 软件的易于使用的环境，相比与MSVC的臃肿，它更加小巧，功能也更强大。

  • 下载 MSYS2 并安装。
  • MSYS2 提供了不同的环境，首先需要决定使用哪个。这些环境之间的主要区别在于环境变量、默认编译器/链接器、架构、使用的系统库等。如果不确定，请选择 UCRT64，下表列出了它们的详细信息：
Name Prefix Toolchain Architecture C Library C++ Library
MSYS /usr gcc x86_64 cygwin libstdc++
UCRT64 /ucrt64 gcc x86_64 ucrt libstdc++
CLANG64 /clang64 llvm x86_64 ucrt libc++
CLANGARM64 /clangarm64 llvm aarch64 ucrt libc++
MINGW64 /mingw64 gcc x86_64 msvcrt libstdc++

Note:

  • Perfix 是 MSYS2 安装目录下的子目录，例如： MSYS2 安装在 C:\msys2，那么 UCRT64 的路径就是 C:\msys2\ucrt64。
  • 当你在msys2中安装软件时，软件会被安装到对应环境的目录下，例如：pacboy -S gcc，软件会被安装到 C:\msys2\ucrt64\ 目录下。
  • MSYS环境包含基于unix-like/cygwin的工具，位于/usr下，且特别之处在于它始终处于激活状态。所有其他环境都继承自MSYS环境，并在其基础上添加各种功能，例如：在UCRT64环境中，$PATH变量以/ucrt64/bin:/usr/bin开头，因此您将获得所有基于ucrt64的工具以及所有msys工具。
  • 打开 MSYS2 UCRT64，如果你使用的不是 UCRT64，请使用对应环境的命令行工具，例如：MSYS2 ClANG64，输入以下命令更新软件包数据库并安装C++编译工具：

Note:

  • pactoys 是MSYS2提供的软件包管理工具，它功能与pacman基本相同，但是它可以直接安装对应环境的软件包，例如：UCRT64环境下pacboy -S gcc等价于pacboy -S gcc。
  • Toolchain 包含了编译工具链，包括：gcc, gdb, make等。
  • cmake 需要单独安装。

2. 配置环境变量与Windows Terminal

  1. 将MSYS2的对应环境bin目录添加到系统环境变量中，例如：C:\msys2\ucrt64\bin。
    2 不建议C:\msys2\ucrt64\usr\bin添加到系统环境变量中，因为该目录下包含的是基于unix-like/cygwin的工具，可能会与系统自带的工具冲突。
    • 这之后，你就可以在命令行中直接使用UCRT64环境下的工具了，例如：gcc --version，gdb --version，nvim等。

3. 配置vscode的C++开发环境

  1. 在 vscode 中安装以下插件：
    1. C/C++：用于C/C++代码的语法高亮、代码补全、调试等功能。
    2. C/C++ Extensions：用于C/C++代码的代码补全、代码导航、代码片段等功能。
    3. CMake Tools：用于CMake项目的构建和调试。
    4. Code Runner：用于运行代码。
  2. 新建一个C++项目，用vscode打开，按Ctrl+Shift+P，选择C/C++: Edit Configurations (UI)，在新页面中，配置如下：
    1. 编译器路径：选择MSYS2的gcc编译器路径，例如：C:\msys2\ucrt64\bin\gcc.exe
    2. 包含路径(include path)：添加MSYS2的include路径，例如：C:\msys2\ucrt64\include\**
    3. 其余按需配置
    • 这会生成一个.vscode目录，其中包含一个c_cpp_properties.json文件，用于配置C/C++项目。
